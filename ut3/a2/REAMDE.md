# UT3-A2 Elige tu IDE ideal

Actividad diseñada para escenario telemático

## Objetivos
- Analizar y comparar distintos entornos de desarrollo. 
- Reflexionar sobre los criterios que influyen al elegir un IDE. 
- Justificar una elección técnica de forma razonada.



## Contexto
Imagina que formas parte de un nuevo equipo de desarrollo que va a iniciar un proyecto web o de aplicaciones.  
Antes de empezar, el equipo debe decidir **qué IDE utilizar**. 
Tú eres la persona encargada de **analizar varias opciones y justificar la elección final** ante el grupo.



## Tareas

### Investigación rápida (15 min)
Busca información sobre **tres IDE distintos** (pueden ser de lenguajes o entornos diferentes). 
Algunos ejemplos:

- Visual Studio Code 
    - ***Elegido 1º:*** Editor de texto ligero y rapido para desarrollo web multilenguaje. Este es gratuito y de codigo abierto. Es bastante personalizable. 

- IntelliJ IDEA
    - ***Elegido 2º:*** Es un IDE completo con el que poder programar en Java, Kotlin, Scala y demas. Lo podemos usar para desarrollo empresarial. Este cuenta con varias versiones, tanto de gratuitas como de pago. Es algo mas exigente que Visual Code pero lo compensa con las potentes herramientas nativas y frameworks.

- Android Studio
    - ***Elegido 3º:*** Es un IDE especializado para el desarrollo de aplicaciones Android nativas y multiplataforma siendo tambien gratuito. Es mucho mas exsigente que los otros pero cuenta con herramientas especificas preintegradas para Android.

### Análisis comparativo (20 min)
Completa una tabla con la siguiente estructura:

| Criterio                                                                         | IDE 1                                | IDE 2                        | IDE 3                                 |
|:---------------------------------------------------------------------------------|:-------------------------------------|:-----------------------------|:--------------------------------------|
| Lenguajes principales que soporta                                                | JavaScript, HTML, CSS, JSON, PHP     | Java, Kotlin, Scala          | Kotlin, Java                          |
| Peso o consumo de recursos                                                       | Ligero                               | Medio                        | Pesado                                |
| Facilidad de instalación                                                        | Sencilla y rapida                    | Estandar                     | Estandar                              |
| Funciones destacadas (depuración, refactorización, integración con Git, etc.) | Extenciones para todas las funciones | Integracion nativa y potente | Herramientas especificas para Android |
| Interfaz / usabilidad                                                            | Limpia, altamente personalizable     | Intuitica, potente           | Especializada en desarrollo movil     |
| Comunidad y soporte                                                              | Enorme y activa                      | Grande y profesional         | Enorme, impulsada por google          |
| Licencia / coste                                                                 | Gratuito                             | Version gratuita y de pago   | Gratuito                              |


### Conclusión y justificación (10 min)
Redacta un texto breve (unas 10 líneas) donde respondas:

- ¿Cuál de los tres IDE recomendarías para tu grupo de trabajo? 
- ¿Por qué crees que se adapta mejor a un proyecto de desarrollo web o de aplicaciones? 
- ¿Qué criterio has valorado más (rendimiento, comodidad, lenguaje, integración, etc.)?

    - Para el proyecto que vamos a realizar recomendaria Visual Studio Code ya que creo que podemos trabajar con su gran versatilidad y ligereza. Podemos trabajar en multiples lenguajes y frameworks sin sufrir de rendimiento en nuestra máquina. Que sea gratuito y de codigo libre ayuda a evitar problemas de costes para el grupo. Las mas importante para el equipo serian la ingredidad y el rendimiento ya que contamos con la flexibilidad y compatibilidad con muchos lenguajes y herramientas junto con poca exigencia que tiene el programa sobre nuestro sistema.
